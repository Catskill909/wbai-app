### Key discoveries during build

- **No og:image on article pages** — confirmed by inspection. Cover images exist only as inline `<img>` tags inside the article body.
- **moreheadlines.php has 3,229 articles** — entire archive in one page. We limit to 20 additional beyond the homepage 6.
- **Article pages render blank on mobile** — root cause: no `<meta name="viewport">` + fixed `840px` desktop layout generated by `document.write()` in `header.js`. Fix: fetch HTML server-side, extract `#leftcontentcontainer`, wrap in mobile CSS, load via `loadData()` with `baseUrl: https://wbai.org/`.
- **Floating bottom chevron blocked content** — moved to AppBar leading icon, which is the standard iOS pattern for slide-up sheets.

## 1. What we're building

A native news feed screen inside the WBAI Flutter app that scrapes and displays articles from wbai.org, with tap-to-read opening the full article in an in-app WebView.

---

## 2. Data source

**No official feed exists.** We scrape the wbai.org homepage (`https://wbai.org/`), which renders up to 6 news tiles with structured CSS classes. This was verified live — the scraper already works (see `news-feed-demo/wbai-news.js`).

### Available fields per article

| Field | Source | Example |
|---|---|---|
| `title` | `.news-tile__title` | "Unhinged Trump Sends Troops To Iran" |
| `date` | `.news-tile__date` | "March 31st, 2026" |
| `author` | `.news-tile__author` | "Bob Hennelly" |
| `category` | `.news-tile__category` | "Global" / "Local" |
| `image` | inline style `--tile-image:url(...)` | `https://wbai.org/images/xxx.jpg` |
| `articleUrl` | `href` on `.news-tile` | `https://wbai.org/articles.php?article=4488` |

### Why the homepage and not moreheadlines.php

`moreheadlines.php` uses a completely different old HTML structure (no CSS classes, plain `<span class="headline">` tags, no images, no categories). The homepage tiles are far richer and already proven.

**Limitation:** The homepage only shows 6 articles at a time. If we want more depth later, we can add a second-pass scraper against `moreheadlines.php` (more effort, less value for v1).

## 5. Detailed spec per file

### 5a. `lib/domain/models/news_article.dart`

```dart
class NewsArticle {
  final String title;
  final String date;
  final String author;
  final String category;
  final String? imageUrl;   // nullable — not all tiles may have an image
  final String articleUrl;
}
```

Plain Dart class, no `Equatable` needed for v1 (list display only, no comparison logic).

---

### 5b. `lib/data/repositories/news_repository.dart`

**Responsibilities:**
- GET `https://wbai.org/` with a browser-style User-Agent header (required — bare Dart http gets blocked)
- Parse response HTML using the `html` package
- Query `.news-tile` elements, extract all 6 fields
- Decode `%20` and strip surrounding quotes from the CSS image URL
- Strip `"by "` prefix from the author span
- Cache result in memory with a timestamp; return cached copy if < 30 minutes old

**Error handling:**
- Network timeout: throw a typed `NewsException` with message
- HTTP non-200: throw `NewsException`
- Parse returns 0 articles (site restructured): throw `NewsException('No articles found — site structure may have changed')`

**No disk caching for v1** — memory cache is sufficient given 30-min TTL and typical app session length.

---

### 5c. `lib/presentation/bloc/news_cubit.dart`

Follows same pattern as `SleepTimerCubit` in the codebase (simple `Cubit`, not full `Bloc`).

**States:**

```dart
abstract class NewsState {}
class NewsInitial extends NewsState {}
class NewsLoading extends NewsState {}
class NewsLoaded extends NewsState {
  final List<NewsArticle> articles;
}
class NewsError extends NewsState {
  final String message;
}
```

**Methods:**
- `fetchNews()` — calls repository, emits Loading → Loaded or Error
- Called automatically on first `NewsPage` build; no manual refresh button for v1

---

### 5d. `lib/presentation/pages/news_page.dart`

**Layout:** `Scaffold` with `AppBar` ("News") + `ListView.builder`

**Each card:**
- Cover image: `CachedNetworkImage` (already in pubspec), 16:9 aspect ratio, `BoxFit.cover`, grey placeholder if null
- Category chip: small pill (blue = `WBAIColors.blue` for Global, grey = `WBAIColors.midGray` for Local) — top-left overlay on image
- Title: Oswald font, 16pt, 2-line max, dark brown
- Date + Author: Poppins 12pt, `WBAIColors.lightGray`
- Card has 1dp border, white background, 8px corner radius — matches app's light mode card style

**Tap behavior:** pushes a `Scaffold` containing `InAppWebView` loaded to `articleUrl`. Reuses `flutter_inappwebview` already in the app. Does NOT reuse `StationWebView` widget directly (that widget is hardcoded to `StreamConstants.stationWebsite`), but mirrors its implementation with a URL parameter.

**Loading state:** `CircularProgressIndicator` centered, same style as `_buildLoadingContainer` on HomePage.

**Error state:** centered text + "Try Again" button that calls `cubit.fetchNews()`.

**Pull-to-refresh:** `RefreshIndicator` wrapping the `ListView` — forces a fresh fetch bypassing the 30-min cache by resetting the timestamp.

### 5f. `lib/core/di/service_locator.dart` change

Add two registrations at the end of `setupServiceLocator()`:

```dart
ServiceRegistry._newsRepository = NewsRepository();
ServiceRegistry._newsCubit = NewsCubit(ServiceRegistry.get<NewsRepository>());
```

Add two static fields and two `if (T == ...)` cases to `get<T>()`.

`NewsCubit` is a singleton so the cache persists for the app session. If the user opens News, closes it, and opens it again within 30 min, they get instant results.

## 7. Caching strategy

| Layer | TTL | Mechanism |
|---|---|---|
| In-memory (NewsRepository) | 30 minutes | `DateTime` field on repository instance |
| Pull-to-refresh | Immediate bypass | Reset timestamp before fetch |
| No disk cache | — | Not needed for v1 |

The 30-minute TTL is appropriate: WBAI publishes articles infrequently (roughly weekly based on observed dates). A shorter TTL would just waste network requests.

## 8. Build order

Build in this sequence to keep each step independently testable:

1. `pubspec.yaml` — add `html` package, run `flutter pub get`
2. `lib/domain/models/news_article.dart` — model only
3. `lib/data/repositories/news_repository.dart` — can be tested standalone with a `main()` print
4. `lib/presentation/bloc/news_cubit.dart` — wire to repository
5. `lib/core/di/service_locator.dart` — register both
6. `lib/presentation/pages/news_page.dart` + `ArticleWebView` widget — UI last
7. `lib/presentation/widgets/app_drawer.dart` — add nav entry

## 9. Design refinements — v1 final

### Visual reference
The WBAI website's own news grid (screenshot provided) is the style target:
- Full-bleed photo cards with a dark gradient overlay at the bottom
- Text (date, title, author) sits over the image bottom-left in white
- Category chip (pill shape) top-right corner in blue (`WBAIColors.blue`)
- Cards have a subtle border and 8px corner radius

### Responsive grid
| Device | Columns |
|---|---|
| Phone (shortestSide < 600) | 1 |
| Tablet (shortestSide ≥ 600) | 2 |

### Entry point
A pill-shaped **"NEWS"** button fixed at the **center bottom** of the home screen (sits between the existing Donate and Sleep Timer circle buttons). Uses `WBAIColors.blue` fill, white text + `Icons.article` icon, same elevation as the side buttons.

### File organization — feature folder
All news code lives under `lib/features/news/` so the entire feature can be removed by deleting one folder + 3 small integration points. This also makes it a clean template for other station apps.

```
lib/features/news/
  models/news_article.dart
  repository/news_repository.dart
  bloc/news_cubit.dart
  widgets/news_card.dart
  pages/news_page.dart
  pages/article_webview_page.dart
```

### Article tap
Tapping a card pushes `ArticleWebViewPage` with the article URL. That page has its own `AppBar` with a standard back arrow and the article title truncated to 1 line.

## 10. What's explicitly out of scope for v1

- `moreheadlines.php` deeper archive scrape
- Full article body text extraction (requires second HTTP call per article)
- Push notifications for new articles
- Offline reading / disk cache
- Search or filtering by category
- Article sharing

These can all be added in v2 once the feed is live and tested.
