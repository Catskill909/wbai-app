## DECISION: LEAVE REMAINING WARNINGS ALONE

**None of these warnings block App Store distribution.** Apple rejects on crashes and functionality failures ‚Äî not deprecated API usage.

**Why patching isn't worth it:**
- Both blocked packages (`flutter_inappwebview_ios`, `audio_service`) are already at their latest available versions. There is literally nothing to upgrade to right now.
- The only workarounds are patching pub-cache directly (fragile, breaks on any `pub get`) or forking the packages (ongoing maintenance burden, drift from upstream). Neither is worth it for cosmetic warnings.
- Apple has left APIs like `initWithImage:` and `keyWindow` deprecated-but-functional for 6+ years. They rarely remove them.

**What to watch:**
- `flutter_inappwebview` 6.2.0 stable ‚Äî one `flutter pub upgrade` will clear ~20 warnings instantly. Subscribe to [pichillilorenzo/flutter_inappwebview releases](https://github.com/pichillilorenzo/flutter_inappwebview/releases) on GitHub.
- `audio_service` 0.19.x ‚Äî will clear the `initWithImage:` warning.

## EASY WINS

### 1. AppIcon ‚Äî 6 Unassigned Children
**Warning:**
```
Runner/Assets.xcassets/AppIcon.appiconset/(null) - The app icon set "AppIcon" has 6 unassigned children.
```

**Root cause:** The `AppIcon.appiconset` folder contains 22 PNG files but `Contents.json` only references 20. The 6 orphaned files are pre-iOS 7 / iPad legacy sizes that Apple no longer requires:

| File | Era |
|---|---|
| `Icon-App-50x50@1x.png` | Old iPad (pre-iOS 7) |
| `Icon-App-50x50@2x.png` | Old iPad (pre-iOS 7) |
| `Icon-App-57x57@1x.png` | Original iPhone (1x) |
| `Icon-App-57x57@2x.png` | iPhone 4 (Retina) |
| `Icon-App-72x72@1x.png` | Old iPad (non-Retina) |
| `Icon-App-72x72@2x.png` | Old iPad (Retina) |

**Fix:** Delete the 6 unassigned PNGs. They are never used by modern iOS (16+).

```bash
cd wbai_radio/ios/Runner/Assets.xcassets/AppIcon.appiconset
rm Icon-App-50x50@1x.png Icon-App-50x50@2x.png \
   Icon-App-57x57@1x.png Icon-App-57x57@2x.png \
   Icon-App-72x72@1x.png Icon-App-72x72@2x.png
```

**Effort:** 1 command. Zero risk.

### 2. Pods ‚Äî "Update to Recommended Settings"
**Warning:**
```
Pods/Pods.xcodeproj ‚Äî Update to recommended settings
```

**Root cause:** Xcode detected that the Pods project's build settings are using outdated defaults (e.g. deprecated linker flags, old Swift/Objective-C settings).

**Fix:** Open Xcode ‚Üí click the yellow warning triangle in the issue navigator ‚Üí click **"Perform Changes"** button that Xcode presents. Xcode migrates all settings automatically. Re-run `pod install` afterwards to keep things in sync.

**Effort:** 2 clicks in Xcode. Zero risk. Does not require any code changes.

> **Note:** These settings are regenerated by CocoaPods on every `pod install`. The migration persists because Xcode writes it into Pods.xcodeproj, which CocoaPods respects on future installs.

### 3. `audio_service` 0.18.18 ‚Äî `initWithImage:` deprecated
**File:** `AudioServicePlugin.m:213`  
**Warning:**
```
'initWithImage:' is deprecated: first deprecated in iOS 10.0 ‚Äî Use -initWithBoundsSize:requestHandler:
```

**Root cause:** The plugin uses `MPMediaItemArtwork initWithImage:` which was deprecated when iOS 10 introduced the request-handler version. It's a cosmetic warning ‚Äî the old API still works.

**Current version:** `0.18.18` ‚Äî **this is the latest version** (published ~11 months ago by ryanheise.com).

**Status:** Stuck. Package is at latest. The author has not fixed this deprecation. No open PR addresses it.

**Options:**
- Wait for `audio_service` 0.19.x or a patch release.
- File an issue / PR at `ryanheise/audio_service` on GitHub.
- Use a `dependency_overrides` patch (complex, breaks on `pub upgrade`).

**Risk if ignored:** None. This API still works and won't be removed until a future iOS SDK major version.

### 4. `flutter_inappwebview_ios` 1.2.0-beta.3 ‚Äî 20+ warnings

## LOW-RISK / MONITOR

### 5. `swift-collections` ‚Äî Sendable conformance (Swift 6 mode)
**File:** `Pods/swift-collections/Sources/DequeModule/Deque+Collection.swift:26`  
**Warning:**
```
Stored property '_storage' of 'Sendable'-conforming struct 'Iterator' has non-Sendable type 'Deque<Element>._Storage'; this is an error in the Swift 6 language mode
```

**Root cause:** In Swift 5 this is a warning. In Swift 6 strict concurrency mode it becomes a compile error. Your project currently compiles in Swift 5 mode, so this is non-blocking.

**Fix:** Either:
1. Wait for `swift-collections` to be updated (it's an Apple open-source package and likely already fixed in a newer version).
2. Check if updating CocoaPods to a newer swift-collections version resolves it: `pod update swift-collections` inside `ios/`.

**Risk if ignored:** Low now. Becomes a blocker only if you switch to Swift 6 strict concurrency mode, which Flutter does not currently require.
